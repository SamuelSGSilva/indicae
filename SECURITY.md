# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| latest (main) | ✅ |

## Reporting a Vulnerability

If you discover a security vulnerability in Indicae, **do not open a public issue.**

Please report it responsibly by contacting the owner directly:

- **GitHub**: [@SamuelSGSilva](https://github.com/SamuelSGSilva)
- **Subject**: `[SECURITY] Brief description of the issue`

### What to include in your report

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### What to expect

- Acknowledgement within **48 hours**
- Status update within **7 days**
- Credit in the fix commit if you wish

## Scope

The following are considered in scope:

- Authentication and authorization bypass
- SQL injection / NoSQL injection
- Cross-site scripting (XSS)
- Sensitive data exposure
- Insecure direct object references (IDOR)
- OAuth flow vulnerabilities

## Out of Scope

- Issues in third-party dependencies (report directly to them)
- Theoretical vulnerabilities without proof of concept
- Social engineering attacks

---

Thank you for helping keep Indicae secure.
